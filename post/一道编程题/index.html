一组连续的正整数1~n,问其中只由0和1组成的数字的个数

题目

一组连续的正整数1~n,问其中只由0和1组成的数字的个数
输入: n, 1 < n <= 10e9
示例
> 输入: 10
> 输出: 2

直觉解法

算法

  1. 遍历1~n
  2. 将每个数字转化为字符串,统计其中’0’和’1’的个数之和
  3. 若二者个数之和为字符串的长度,则该数字只由0和1组成,统计变量加1
  4. 遍历完成后输出统计结果

while 1:
    n = input()
    result = 0
    for i in range(1, n+1):
        if str(i).count('0') + str(i).count('1') == len(str(i)):
            result += 1
    print result

效率相同的方法:利用正则表达式

将上述代码中 if 后的语句更换为 re.match(r'[0-1]+$',str(i)) 即可

问题

  1. n的位数每增加一位, 符合题意的数字数量相对就越少(若n=10000,则符合题意的数字只有17个)
  2. 在这种情况下,遍历所有数字效率极低
  3. 而且将会导致内存超限,无法计算出题目中n所有取值的结果

规律

设对于给定的数n,其有k位.将问题分解成两部分:

  • 求前10^(k-1)-1个数中符合题意的数字个数
  • 求10^(k-1)~n中符合题意的数字个数
  • 两个问题的解之和为答案

求前10^(k-1)-1个数中符合题意的数字个数

既然是只包含0和1的数字,那长得就和二进制数完全一样了,那么从二进制数的角度思考一下: 首先问题转换为给定一个位数k,求位数小于等于k的二进制数的个数

观察上方的表格,将包含的数字视为二进制数:

  • n=1时,只包含一个数字1符合条件;
  • n=2时:
    • 符合条件的数字是:1, 10, 11
    • 1是n=1时符合条件的数字
    • 10是1左移1位得到的数字
    • 11是10+1的到的数字
    • 11不能再做处理(否则位数就不符合题意)
  • n=3时:
    • 符合条件的数字是:1, 10, 11, 100, 101, 110, 111
    • 1,10,11是n=2时符合条件的数字
    • 100是10左移1位得到的数字, 101是100+1得到的数字
    • 110是11左移1位得到的数字, 111是110+1得到的数字

发现规律了吗? f(n) = f(n-1) + (f(n-1)-f(n-2)) * 2

求10^(k-1)~n中符合题意的数字个数

这个问题的关键在于用所给的数n与一些特定的数作比较,而这些特定的数的规律同上

def getPre(k):
    if k==0: return 0
    if k==1: return 1
    return getPre(k-1) + ( getPre(k-1) - getPre(k-2) ) * 2

def getCur(k,n):
    if k==1: return 1
    count = 0
    flag = 10**(k-1)
    for i in range(k):
        if n>=(flag+1): count=count+2
        elif n>=flag: count=count+1
        else: break
        flag = flag + 10**(i+1)
    return count

while 1:
    n = int(input())
    k = len(str(n))
    print(getPre(k-1) + getCur(k,n))
